# 10. Procedural Generation

## 1. 목표 — LOCKED

맵은 매번 달라야 하지만, 목표 배치와 이동 가능성은 검증 가능해야 한다.

## 2. 2단계 생성 구조 — LOCKED

### 1단계: 거시 그래프

먼저 기능적 관계를 생성한다.

- 입구
- 주요 목표
- 보스 또는 네임드 후보 구역
- 선택 보물방
- 서브 던전 입구 후보
- 귀환 경로
- 우회로

### 2단계: 미시 지형

거시 그래프를 실제 타일 지형으로 변환한다.

- 암석 경도
- 광맥
- 공동
- 비밀방
- 적 둥지
- 장식
- 붕괴 가능 지층

## 3. 생성 보장 규칙

- 입구에서 핵심 목표까지 최소·최대 거리
- 최소 하나 이상의 유효 경로
- 채굴 도구 기본 성능으로 진행 가능
- 보스방에 필요한 최소 전투 공간
- 탈출 구간이 완전히 봉쇄되지 않음
- 서브 던전 입구가 벽 안에 묻히지 않음
- 필수 자원이 접근 불가능한 셀에 생성되지 않음

## 4. 검증기 — LOCKED

생성 후 자동 검증 단계를 둔다.

검증 항목:

- 경로 연결성
- 목표 접근 가능성
- 보스 공간 크기
- 탈출 경로 수
- 위험 요소 중첩
- 자원 밀도
- 네임드 출현 위치

검증 실패 시 시드를 폐기하거나 해당 구역만 재생성한다.

## 5. 지형 상호작용 범위

### MVP 포함

- 셀 파괴
- 경도 차이
- 폭발 범위 파괴
- 비밀방
- 제한적 붕괴 또는 봉쇄
- 적의 벽 파괴

### 후순위

- 대규모 지형 붕괴
- 실시간 물 흐름
- 실시간 용암 확산
- 복잡한 압력·가스 시뮬레이션

환경 위험은 구역·타이머·상태 기반으로 단순화하는 방향을 우선한다.

## 6. 시드와 재현성

- 모든 원정 맵은 시드로 재현 가능해야 함
- 버그 리포트에 시드 저장
- 생성 버전 번호를 세이브와 함께 기록
- 패치 후 동일 시드가 달라질 수 있음을 허용하되 디버그 빌드는 생성 버전 고정 가능
