# Mining Survival Roguelite GDD v0.1

> 작업명: **Project Deep Seal** (가칭)  
> 문서 목적: 1인 개발을 위한 게임 기획 기준선과 변경 가능한 설계 토대  
> 문서 상태: 초안 / 지속 갱신형  
> 주요 플랫폼: Steam / PC  
> 플레이 방식: 싱글플레이

## 1. 프로젝트 한 줄 정의

여러 광부를 반복해서 절차적으로 생성되는 심층 지하에 보내고, 자동 전투와 방향성 채굴로 자원·보물·봉인석을 확보하며, 부상·실종·구조 실패의 위험을 관리한 끝에 최종 심층부의 핵심 유물을 훔쳐 탈출하는 원정대 운영형 로그라이트.

## 2. 이 문서 세트의 사용법

이 문서는 완성된 사양서가 아니라 **변경 가능한 설계 기준선**이다. 각 요소는 다음 상태 태그로 관리한다.

- **LOCKED**: 현재 기획의 핵심으로 합의된 요소. 변경 시 다른 시스템에 미치는 영향 검토 필요.
- **DRAFT**: 방향은 유효하지만 수치·규칙·콘텐츠 규모가 미확정인 요소.
- **DEFERRED**: 핵심 재미 검증 후 도입 여부를 판단할 요소.
- **OUT**: 현재 범위에서는 명시적으로 제외한 요소.

새 시스템을 추가할 때는 각 문서의 템플릿에 따라 다음을 적는다.

1. 설계 목적
2. 플레이어가 내리는 결정
3. 입력과 출력
4. 다른 시스템과의 의존성
5. 실패 가능성과 예외 상황
6. 조정 가능한 수치
7. 현재 상태

## 4. 현재 확정된 핵심 방향

- **LOCKED** Steam PC 싱글플레이
- **LOCKED** Vampire Survivors 계열의 거의 완전한 자동 전투
- **LOCKED** 여러 번의 원정을 통한 캠페인 구조
- **LOCKED** 절차적 생성 맵과 방향성 채굴
- **LOCKED** 보물·봉인석 수집 후 최종 심층부 공략
- **LOCKED** 원정 실패 시 대부분 부상 또는 실종
- **LOCKED** 실종 후 구조 실패, 고위험 원정 실패 등에서 영구 사망
- **LOCKED** 원정 전 정보와 준비, 현장 대응, 후퇴 판단을 난도의 중심으로 사용
- **LOCKED** 네임드 엘리트·보스는 상성에 따라 난도가 크게 달라지되 하드 카운터는 지양
- **LOCKED** 일반 원정 중 무작위 서브 던전 등장
- **LOCKED** 2D 타일 기반 구조에 2.5D 인상을 주는 비주얼 우선 검토
- **OUT** 멀티플레이
- **OUT** 복잡한 유체 시뮬레이션을 핵심 시스템으로 사용
- **OUT** 다수의 생존 게이지를 동시에 관리하는 하드코어 생존 게임

## 5. 기획 변경 규칙

핵심 시스템을 수정할 때는 다음 항목을 함께 갱신한다.

- 관련 문서의 상태 태그
- `docs/12_open_questions.md`
- `docs/13_decision_log.md`
- MVP 범위에 영향을 주면 `docs/11_scope_mvp_roadmap.md`

문서를 코드보다 먼저 갱신하는 것을 원칙으로 하되, 프로토타입 검증 결과가 문서와 충돌하면 **재미와 구현 가능성**을 우선하고 문서를 즉시 수정한다.
