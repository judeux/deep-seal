# 13. Decision Log

## 2026-06-12 — 플랫폼과 모드

- 결정: Steam PC를 우선 플랫폼으로 한다.
- 결정: 싱글플레이 전용으로 기획한다.
- 이유: 1인 개발 범위, 파괴형 절차적 맵의 동기화 비용, 캠페인 중심 설계.

## 2026-06-12 — 전투 방식

- 결정: Vampire Survivors 계열의 거의 완전한 자동 전투를 사용한다.
- 플레이어의 주요 조작은 이동, 채굴, 성장 선택, 제한된 비상 액티브다.

## 2026-06-12 — 캠페인 구조

- 결정: 한 번의 원정에서 게임을 완결하지 않는다.
- 여러 번의 원정으로 봉인석을 축적하고 최종 심층부를 여는 캠페인 구조를 사용한다.
- 참고 감각: Darkest Dungeon식 장기 원정대 운영.

## 2026-06-12 — 아트 방향

- 결정: 1인 개발과 에셋 활용을 고려해 2D 타일 기반 구조를 우선한다.
- 방향: 2D 조명과 가짜 높이를 사용해 2.5D 쿼터뷰 인상을 구현한다.
- 완전 3D 파괴형 지형은 현재 범위에서 제외한다.

## 2026-06-12 — 실패와 영구 사망

- 결정: 일반 원정 실패의 기본 결과는 부상 또는 실종이다.
- 결정: 실종 후 구조 실패, 봉인석·최종 등 고위험 원정 실패에서 영구 사망이 발생한다.
- 이유: 자동 전투의 통제 한계를 고려하면서 손실의 무게를 유지하기 위함.

## 2026-06-12 — 난도와 상성

- 결정: 원정 전 준비와 보스·네임드 상성이 난도를 크게 바꾸게 한다.
- 결정: 특정 장비가 없으면 절대 승리할 수 없는 하드 카운터는 지양한다.
- 결정: 준비 부족 시 현장 보완, 우회, 후퇴가 가능해야 한다.

## 2026-06-12 — 서브 던전

- 결정: 일반 원정 중 무작위 선택형 서브 던전을 등장시킨다.
- 방향: 각 서브 던전은 짧고 명확한 고유 규칙과 고유 보상을 가진다.

## 2026-06-12 — 복잡한 환경 시뮬레이션

- 결정: 낮은 곳으로 흐르는 물, 연속적으로 확산하는 용암 등 복잡한 유체 시뮬레이션은 핵심 범위에서 제외한다.
- 대안: 구역, 타이머, 상태 단계로 직관적으로 표현한다.
